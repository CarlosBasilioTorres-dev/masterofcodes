Variables en Java

Publicado el 9 de mayo de 2025

Bienvenido de nuevo.

Hasta este punto del curso, únicamente hemos creado un programa muy simple: el "Hola, mundo". Espero que la explicación proporcionada haya sido de utilidad.

Pasemos ahora a temas más interesantes.

Si bien escribir mensajes en una aplicación es de mucha utilidad, un programa que solo muestra mensajes resulta inservible en muchos casos. Lo importante, en el 99.99 % de los casos, es manejar información.

¿Cómo almacenar información en Java?

Java nos proporciona 3 formas básicas de almacenar información dentro de un código fuente:

  1. Variables
  2. Constantes
  3. Variables con inferencia de tipo

Independientemente de la forma que elijas, la estructura básica es la misma: 

tipo_de_dato nombre valor;

Aquí hay muchas cosas que explicar; te recomiendo que revises el Manual de operaciones básicas de Java y el Manual de tipos de dato para entender algunos conceptos que aquí solo voy a mencionar.

Tipo de dato

Se refiere al tipo de información que vas a guardar (enteros, flotantes, double,boolean, etc).

Nombre

Es el nombre con el cual tu vas a identificar tu información.

El nombre que elijas para tu variable debe cumplir algunas reglas.

  1. Debe comenzar con una letra o guion bajo (_)
  2. Despues de la primer letra, puedes colocar numeros, letras o guiones bajos
  3. (Regla camel case) La primer letra debe ser minúscula
  4. No debe contener espacios, ni caracteres especiales, salvo _

Ejemplos validos

  1. var_1
  2. _var_1
  3. segunda_variable

Ejemplos invalidos

  1. 1_var
  2. variable-1
  3. 2da_variable

Valor

Opcionalmente, puedes inicializar tu literal con algún valor (excepto en constantes, ahí es obligatorio).

Si te das cuenta las reglas son muy simples, solamente hay que tener cuidado al momento de utilizarlas. Una vez entendido esto, continuamos con la explicación.

Variables

Comencemos a programar un poco, crea un nuevo archivo "Variables.java" y escribe lo siguiente: 

public class Variables {
    public static void main(String[] args) {
        int edad = 33;
        System.out.println("Mi edad es: " + edad);
    }
}

Si todo est bien, obtendrás el siguiente resultado:

Mi edad es: 33
La definición más clara que puedo dar de una variable es la siguiente:

"Una literal que almacena una dirección de memoria en la cual asignaremos un valor que puede cambiar a lo largo del tiempo."

¿Entendiste? No te preocupes, yo tampoco. Vamos más despacio.

Imagina tu clase de matemáticas de la prepa, en la cual te decían lo siguiente: 

x + 5 = 10  => x = 5
Si x + 5 es igual con 10, esto implica que x es igual a 5

Si te das cuenta, x es una literal (una letra) a la cual le asignamos un valor (en este caso, 5). En programación es lo mismo: una literal a la cual le asignamos un valor. Pero, en este caso, el valor que le asignamos es una dirección de memoria (un lugar en la memoria RAM) en el cual se almacena la información que le indiquemos (en el caso del programa, 33).

Este tipo de almacenamiento recibe el nombre de Variable por que, como su nombre lo indica, es un valor cambiante en el tiempo (en una parte del código vale una cosa y en otro otra). Vamos a dejarlo más claro con un ejemplo:

Aumentemos a nuestro código algunas lineas:

public class Variables {
    public static void main(String[] args) {
        int edad = 33;
        System.out.println("Mi edad es: " + edad);
            
            
        edad = 38;
        System.out.println("Mi edad es: " + edad);
    }
}

Tu salida debe ser la siguiente:

Mi edad es: 33
Mi edad es: 38

Si lo revisas con detenimiento verás que edad en la primera concatenación valía 33. Para la segunda, cambiamos su valor a 38, es decir, en un momento en el tiempo valía 33, pero ahora vale 38.

Constantes

Misma idea inicial que en variables, solamente que, como su nombre lo indica, el valor no puede cambiar con el tiempo, es decir, debe ser constante durante toda la ejecución.

Retomando el ejemplo anterior, haremos una modificación: 

public class Variables {
    public static void main(String[] args) {
        final int edad = 33;
        System.out.println("Mi edad es: " + edad);
        
        
        edad = 38;
        System.out.println("Mi edad es: " + edad);
    }
}

Este código debe mostrar el siguiente error: 

java: cannot assign a value to final variable edad

El error es muy explícito

No puedes asignar un valor a la variable final edad

Como su misma declaración lo explica, edad tiene un valor entero final (no puede cambiar). Al redefinir su valor a 38, entonces es cuando esto intenta violar el principio básico de la contante y es cuando marca el error.

Variables con inferencia de tipo:

Se recomienda leer el Manual de tipos de dato antes de esta sección.

A partir de Java 10, se incluye una nueva forma de declarar variables en java, el cual tiene un uso muy particular: Variables por inferencia de tipo.

Sin darle muchas vueltas al asunto, esto implica que, dado un dato, Java es capaz de identificar el tipo de dato y por consiguiente, trabajarlo como lo que él detecte.

Vamos a un ejemplo: 

public class Variables {
    public static void main(String[] args) {
        var edad = 33;
        System.out.println("Mi edad es: " + edad);


        edad = 38;
        System.out.println("Mi edad es: " + edad);
    }
}

Lo anterior devuelve la misma respuesta que el programa anterior (sin errores), pero funciona con var (definición que indica que trabajamos con una variable de inferencia).

Todo adelanto tiene un precio

Este tipo de variables no infieren su tipo de dato en el proceso de interpretación (es decir en ejecución), lo hace en el tiempo de compilación. Lo anterior implica que, al momento de declarar la variable, debemos de inicializarla para que pueda funcionar.
