Operaciones básicas en Java

En Java, existen 4 operaciones básicas que no requieren nada adicional a lo creado en el Hola mundo, estas operaciones son:

  1. Suma
  2. Resta
  3. Multiplicación
  4. División

Suma

Aunque parezca lógico, en este curso no damos nada por hecho. Hagamos un programa simple para entender la suma de 2 números en java. 

public class ManualOperacionesBasicas {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;
        int c = a + b;
        System.out.println(a + " + " + b + " = " + c);
    }
}

El resultado del programa es el siguiente: 

10 + 20 = 30

Las tres primeras lineas son simples de entender si leíste el manual de Tipos de dato en Java:

  1. Declaramos una variable entera de nombre "a" con valor de 10
  2. Declaramos una variable entera de nombre "b" con el valor de 20
  3. Declaramos una variable entera de nombre "c" cuyo valor será la suma de a y b

El método println es otra cosa.

Concatenación

El operador suma (+) dentro de Java, nos indica una operación adicional a la adición, y es denominada concatenación. Esta operación consiste en unir 2 valores (texto, números o variables) como parte de un solo texto.

Pensemos un minuto en el println usado en el código anterior: 

System.out.println(a + " + " + b + " = " + c);

Lo que estamos diciendo en esa linea en un lenguaje completamente informal es lo siguiente:

  1. System.out.println: Voy a imprimir algo y al final de la impresión haces un salto de linea. (ln = enter en un procesador de texto).
  2. a: Muestra el valor de la variable a
  3. + " + ": en seguida vas a mostrar el texto +
  4. + b: después del + muestra el valor de b
  5. + " = ": posteriormente vas a mostrar el símbolo de =
  6. + c y por último vas a mostrar el valor de c

En resumen, la concatenación nos permite unir texto y valores de variables, lo cual se hace muy util al momento de mostrar resultados a los usuarios.

División

public class ManualOperacionesBasicas {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;
        int c = a / b;
        System.out.println(a + " / " + b + " = " + c);
    }
}

Resultado:

10 / 20 = 0

¿Que paso aquí? El problema es el tipo de dato

Relájate, todo lo hicimos bien (mas o menos). El resultado obtenido en lógica computacional es correcto. Aquí de explico el por que:

Si revisas el código con detenimiento, veras que todas las variables son enteras, es decir, solamente pueden guardar números enteros. Ahi esta la clave del problema.

Debido a que C almacena una división no entera (en este caso .5), Java redondea el resultado al entero mas cercano hacia abajo por defecto, es decir, si el resultado es .5, Java almacenará 0 (si fuera 1.5 almacenaría 1, etc).

La solución

Veamos que pasa si almacenamos ahora el resultado en una variable float:

public class ManualOperacionesBasicas {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;
        float c = a / b;
        System.out.println(a + " / " + b + " = " + c);
    }
}

Si hacemos esto, mi editor muestra un warning (no es un error, el programa va a funcionar pero quizá no como lo esperamos) que dice lo siguiente:

'a/b' : integer division in floating-point context

Y el resultado vuelve a ser erróneo

10 / 20 = 0.0

Traducirlo al español no sirve de nada si no entendemos el contexto del que nos habla, vamos a explicarlo detenidamente:

Lo que nos indica esta advertencia es que, dado que estamos operando números enteros, Java determina que la respuesta debe ser entera, por lo cual hace lo mismo que en el caso anterior, redondea hacia abajo y nos muestra como resultado 0.

En resumen, Java nos esta diciendo:

Si tengo 2 números enteros, por mis (introduzca su insulto favorito)
el resultado va a ser entero.

¿Ahora si hay solución?

Si, de hecho tenemos 2 formas de solucionarlo:

Opción 1: Cambia una variable a flotante 

public class ManualOperacionesBasicas {
    public static void main(String[] args) {
        int a = 10;
        float b = 20;
        float c = a / b;
        System.out.println(a + " / " + b + " = " + c);
    }
}

Opción 2: Haz un cast al resultado:

public class ManualOperacionesBasicas {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;
        float c =(float) a / b;
        System.out.println(a + " / " + b + " = " + c);
    }
}

Hacer un "cast" implica que estableceremos un contexto en el cual vamos a trabajar. En este caso en particular, le indicamos que, aunque sean 2 enteros, queremos que a se comporte como un decimal para efectos de la operación.
