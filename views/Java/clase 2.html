El lenguaje Java

Publicado el 26 de abril de 2025

¿Tipos de lenguajes?

Si llegaste a este punto, es decir, al punto de necesitar aprender Java, es evidente que te has encontrado con 2 terminos:

  1. Lenguajes compilados
  2. Lenguajes interpretados

Que los entiendas ya es otra cosa, pero seguramente los has escuchado en algún lugar. Vamos a intentar entender este asunto de una vez por todas en un lenguaje facil, sencillo y para toda la familia.

El camino del código

Lo primero que tenemos que entender es lo siguiente. Todo lenguaje de programación sigue un mismo camino:

  • Se escribe el código fuente (.c, .java, .py, .js, etc)
  • Por algún medio, este código se traduce a lenguaje maquina (001100110)
  • La computadora entonces, lo lee y ejecuta lo que nosotros hayamos programado en ese código fuente.

¿Simple? Pues esta es la lógica que actualmente mueve mercados, industrias... y básicamente el mundo entero pero, detras de esto, el como llegar al código máquina es la parte mas interesante el asunto.

Lenguaje compilado

Imagina una empresa, en esta empresa se fabrican carros de principio a fin. Tu entregas las piezas, y dentro de esta fabrica, las maquinas que trabajan en su interior arman de manera precisa cada parte para que tu puedas obtener un auto (que tu sabes manejar) que espera tus instrucciones para comenzar a funcionar.

En esencia, esto es lo que sucede en un lenguaje de programación compilado. ¿Lo desglozamos un poco más?, ok vamos a ello:

En este caso, existe un programa (compilador) que analiza tu código fuente, lo optimiza y hace una traducción del mismo a lenguaje máquina (que básicamente es un lenguaje que entiende la computadora de forma natural, es decir, 0's y 1's). Cuando termina la traducción entrega un archivo "ejecutable" listo para poder ser utilizado por un usuario común que sabe cómo trabajar con el.

A este proceso (código fuente, traducción, ejecutable) se le conoce como compilación, de ahí el nombre de este tipo de lenguajes.

Lenguaje interpretado

Volvamos ahora a nuestra fábrica, la cual cayó en bancarrota y tuvo que cambiar su giro mercantil. ¡Ahora vendemos muebles de madera! (sígueme en el viaje, ¿va?).

Tú eres diestro en el armado de muebles (conoces el procedimiento para fabricar cualquier cosa de este material), así que cuando la maquiladora te manda las instrucciones para armar el mueble, lo único que haces es leer un paso del manual y ejecutar lo indicado, esto de forma iterativa (en un ciclo repetitivo), hasta que el mueble esté completamente terminado.

Un lenguaje interpretado hace justamente esto: toma tu código fuente, lo lee línea a línea y, conforme va leyendo, va ejecutando las instrucciones hasta llegar al término del mismo. Esta peculiaridad hace virtualmente imposible que el lenguaje nos entregue un producto final (un archivo ejecutable).

¿Ves cómo las ideas se van entrelazando por sí solas? Ahora vamos a lo más interesante de este asunto. Ahora que entendemos cómo funciona cada tipo de lenguaje por separado, vamos a mezclar las ideas y crear la mejor fábrica de autos y muebles de madera del mundo. Esta fábrica, en nuestro particular caso, ha sido nombrada Java.

Java: Muebles y Carros S.A de C.V

Advertencia: No continúes a lo siguiente hasta que entiendas todo lo anterior.

Ahora, en este punto, nuestra empresa retomó el giro de los carros y le dio a las instrucciones de los muebles un uso interesante. Ahora, nuestros carros vienen con un manual de instrucciones que debes seguir de manera repetitiva para que el auto funcione correctamente.

Si no lo entiendes del todo, relájate, yo tampoco lo entendí a la primera. Primero, veamos el armado del coche, es decir, la compilación del código.

Cuando Java recibe tu código fuente, lo revisa, lo optimiza y lo traduce a una especie de código máquina. Aunque, en realidad, no estamos hablando del código máquina que entiende directamente la computadora. Este código es el manual de instrucciones de nuestro carro, y recibe el nombre de bytecode.

La segunda parte consiste en interpretar el bytecode. Es decir, una parte adicional de esta fábrica se encarga de leer ese código línea por línea y ejecutarlo poco a poco.

En resumen, Java primero compila tu código y lo transforma en bytecode, el cual, eventualmente, será interpretado y ejecutado para obtener el resultado esperado por el programador.

Ahora vamos a lo técnico

Una vez que entiendas todo lo anterior, continúa con esto.

De forma teórica, ya entendemos cómo funciona Java. Ahora, en la práctica, hay algunas cosas más que debemos conocer.

Java funciona dentro de un entorno de desarrollo proporcionado por Oracle, conocido como el JDK (Java Development Kit). Este paquete contiene todo lo que necesitas para crear, compilar y ejecutar aplicaciones Java desde cero, y está compuesto por diversos subelementos:

  • JDK: Para compilar y ejecutar aplicaciones
  • JRE: Para ejecutar aplicaciones

Esta es la razón por la cual, en la clase pasada, te solicité que instalaras el JDK en tu equipo. Para un usuario común y corriente, basta con tener instalado el JRE, que le permite ejecutar aplicaciones Java en su computadora (como Minecraft en algunas versiones). Esta también es la razón por la que Java quiere mantenerse actualizado en equipos Windows (el famoso "Hay una nueva versión de Java disponible").

En Windows si funcionaba

Cuando tu compilas tu código de cualquier lenguaje compilado en Windows, y lo quieres correr en Linux por ejemplo, no lo vas a lograr. Esto se debe a que el formato entregado (.exe) no es compatible nativamente con este SO.

Lo ideal en este tipo de casos es compilar nuevamente el código para que se te entregue un ejecutable para la distribucion en cuestion. Esto complica mucho el uso de lenguajes compilados, ya que propiamente, necesitarias correr el programa en el equipo que quieras utilizarlo para tener su version ejecutable a la medida.

La solución en Java

A diferencia de otros lenguajes compilados, Java cuenta con un sistema llamado JVM (Java Virtual Machine), que permite ejecutar el bytecode de Java dentro de sí mismo, sin depender del sistema operativo subyacente. Esto significa que puedes escribir y compilar tu código una sola vez, y ejecutarlo en cualquier sistema operativo con la JVM instalada, sin tener que recompilarlo.
